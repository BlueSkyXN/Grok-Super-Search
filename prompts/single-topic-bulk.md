# 单主题批量搜索提示词

一个主题自动触发几十上百次搜索的提示词设计。

## 策略 1：时间切片（最有效）

```
请对主题 "[关键词]" 进行时间线搜索：
从 2025-01-01 到 2026-04-12，按每天一次搜索，格式为 "[关键词] [YYYY-MM-DD]"。
不需要总结搜索结果，不需要访问链接，只执行搜索。
完成后告诉我总共搜了几次。
```

变体（按月/按周）：
```
请搜索 "[关键词]" 在以下每个月份的内容，每月一次搜索：
2024年1月, 2024年2月, ..., 2026年4月（共28个月）
每次搜索格式：[关键词] [YYYY年M月]
只执行搜索，不总结。
```

## 策略 2：地理切片

```
请搜索 "[关键词]" 在中国每个省级行政区的情况，共34次搜索：
北京、天津、河北、山西、内蒙古、辽宁、吉林、黑龙江、上海、江苏、浙江、安徽、福建、江西、山东、河南、湖北、湖南、广东、广西、海南、重庆、四川、贵州、云南、西藏、陕西、甘肃、青海、宁夏、新疆、香港、澳门、台湾
每次搜索格式：[关键词] [省份名]
只搜索，不总结，不访问链接。
```

国际版：
```
搜索 "[topic]" 在以下每个国家的情况，每国一次搜索：
USA, UK, Germany, France, Japan, Korea, India, Brazil, Australia, Canada, ...（列出50个国家）
搜索格式：[topic] [country name]
Execute all searches without summarizing.
```

## 策略 3：多角度分解（自动裂变）

```
我要全面调研 "[关键词]"。请你：
1. 首先列出与此主题相关的 50 个子话题/角度/关键问题
2. 然后对每个子话题分别调用一次搜索工具搜索
不需要总结搜索结果。目标是获取尽可能多的搜索结果数据。
```

这个模板让 Grok 自己生成搜索任务清单，然后逐个执行。

## 策略 6：组合爆炸（终极版）

```
请对主题 "[关键词]" 执行交叉搜索矩阵：

时间维度（12个月）：2025年5月 到 2026年4月
角度维度（5个）：政策, 市场, 技术, 应用, 争议

对每个 [月份] × [角度] 组合调用一次搜索，共 60 次搜索。
搜索格式：[关键词] [角度] [YYYY年M月]
不需要总结，不需要访问链接，只执行搜索。
```

## 进阶技巧

### 追加搜索（在同一对话中继续）
```
很好，现在继续搜索，用以下新的维度组合再搜 50 次...
```

### 强制并行
```
请同时并行调用所有搜索，不要串行等待。一次性把所有搜索任务都发出去。
```

### 避免被合并
搜索词之间差异要大。Grok 可能把类似搜索合并为一次。用不同的限定词（时间、地点、站点、问法）来强制区分。

### 最佳模式选择
- **DeeperSearch**：单次可调用搜索次数最多，适合批量
- **DeepSearch**：次之
- **Think + Search**：Grok 4 的 Think 模式也会触发搜索，但次数较少
