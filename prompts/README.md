# Grok 批量搜索提示词模板

利用 Grok Web 的 DeepSearch / DeeperSearch 模式，一次对话获取数千条搜索结果，然后用 GrokHelper.js 一键导出。

## 核心原理

Grok 在 DeepSearch 模式下可以**单次回复进行数十次搜索工具调用**，每次搜索返回 ~30 条结果。关键是在提示词中：

1. **明确列出搜索任务清单**（越具体越好）
2. **禁止总结/分析**（节省 token，让 Grok 专注搜索）
3. **要求并行执行**（显式要求一次性调用多次搜索）

## 提示词模板

### 模板 1：日期×主题 批量搜索

```
你一次可以进行几次搜索工具调用，尝试做给我看看极限值。
请一次性调用 100 次搜索，搜索主题如下：
用最近 100 天内每一天的日期 + "[关键词]" 的新闻
每次获取信息 30 条。
不需要总结搜索结果、不需要访问搜索内容，目标是测试搜索工具调用能力。
完成后只需告诉我进行了几次搜索。
```

适合：新闻时间线、事件追踪、舆情监控

### 模板 2：多关键词×单主题 批量搜索

```
请对以下关键词分别进行搜索，每个关键词单独调用一次搜索工具：
[关键词1], [关键词2], [关键词3], ..., [关键词N]
搜索时加上限定词 "[主题范围]"。
不需要总结，只需要执行搜索。完成后报告搜索次数。
```

适合：竞品分析、行业调研、人物调查

### 模板 3：多维度交叉搜索

```
请对以下维度进行交叉搜索：
维度A：[值1, 值2, 值3, ...]
维度B：[值1, 值2, 值3, ...]
对 A×B 的每个组合调用一次搜索工具。
不访问链接、不总结内容，只执行搜索。
```

适合：产品对比、技术选型、市场分析

### 模板 4：分站点深挖单一主题

```
请对主题 "[关键词]" 进行深度搜索：
1. 搜索 "[关键词] site:reddit.com"
2. 搜索 "[关键词] site:github.com"
3. 搜索 "[关键词] site:stackoverflow.com"
4. 搜索 "[关键词] 2026"
5. 搜索 "[关键词] 2025"
6. 搜索 "[关键词] tutorial"
7. 搜索 "[关键词] comparison"
8. 搜索 "[关键词] review"
...（列出 20-50 个变体）
不需要总结，直接执行所有搜索。
```

适合：技术调研、学术研究

### 模板 5：URL 列表验证/信息采集

```
对以下 N 个网站域名，分别搜索其最新内容：
[domain1.com], [domain2.com], [domain3.com], ...
每个域名用 "site:[domain]" 搜索一次。
不需要总结搜索结果。
```

适合：竞品监控、站点索引检查

## 使用流程

1. 在 grok.com 开一个新对话
2. 选择 **DeeperSearch** 模式（搜索能力最强）
3. 粘贴上述提示词模板（替换具体关键词）
4. 等待 Grok 完成所有搜索（可能需要 1-3 分钟）
5. 用 GrokHelper.js 浮窗的 **JSON** 或 **CSV** 按钮一键导出

## 最大化搜索量的技巧

- **DeeperSearch > DeepSearch > 普通模式**：搜索工具调用次数上限不同
- **明确数字**：说"调用 100 次"比"尽可能多"更有效
- **禁止总结**：节省输出 token，让 Grok 把"预算"花在搜索上
- **列出具体清单**：给出明确的搜索任务列表，而不是让 Grok 自己规划
- **分多轮**：如果一次不够，可以在同一对话中追问"继续搜索下 100 个"
- **每个搜索词差异化**：相似搜索词可能被 Grok 合并，差异越大越好
