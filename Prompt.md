# Grok 批量搜索提示词

利用 Grok DeepSearch/DeeperSearch 一次对话获取数千条搜索结果，配合 GrokHelper.js 一键导出 JSON/CSV。

---

## 核心原理

Grok 在 DeeperSearch 模式下单次回复可进行数十次搜索工具调用，每次返回 ~30 条结果。提示词设计要点：

1. **明确列出或让 Grok 自动生成搜索任务清单**
2. **禁止总结/访问链接** — 节省 token，专注搜索
3. **要求并行执行** — 一次性发出所有搜索

---

## 提示词模板

### 1. 时间切片（最稳定，每天一搜）

```
请对主题 "[关键词]" 进行时间线搜索：
从 2025-01-01 到 2026-04-12，按每天一次搜索，格式为 "[关键词] [YYYY-MM-DD]"。
不需要总结搜索结果，不需要访问链接，只执行搜索。
完成后告诉我总共搜了几次。
```

### 2. 地理切片（34省/多国）

```
请搜索 "[关键词]" 在中国每个省级行政区的情况，共34次搜索：
北京、天津、河北、山西、内蒙古、辽宁、吉林、黑龙江、上海、江苏、浙江、安徽、福建、江西、山东、河南、湖北、湖南、广东、广西、海南、重庆、四川、贵州、云南、西藏、陕西、甘肃、青海、宁夏、新疆、香港、澳门、台湾
每次搜索格式：[关键词] [省份名]
只搜索，不总结。
```

### 3. 多角度自动裂变（最省力，推荐）

```
我要全面调研 "[关键词]"。请你：
1. 首先列出与此主题相关的 50 个子话题/角度/关键问题
2. 然后对每个子话题分别调用一次搜索工具搜索
不需要总结搜索结果。目标是获取尽可能多的搜索结果数据。
```

### 4. 站点遍历

```
请对主题 "[关键词]" 在以下每个网站上搜索一次：
site:zhihu.com, site:weibo.com, site:douban.com, site:bilibili.com,
site:baidu.com, site:36kr.com, site:huxiu.com, site:juejin.cn,
site:csdn.net, site:cnblogs.com, site:reddit.com, site:twitter.com,
site:youtube.com, site:medium.com, site:github.com, site:stackoverflow.com,
site:wikipedia.org, site:arxiv.org
（共18次搜索）
搜索格式：[关键词] site:[域名]
只执行搜索，不总结。
```

### 5. 问题模板裂变

```
请对 "[关键词]" 分别用以下问句搜索（每种问法搜一次）：
"[关键词] 是什么", "[关键词] 怎么用", "[关键词] 优缺点",
"[关键词] 对比", "[关键词] 替代方案", "[关键词] 最新进展",
"[关键词] 价格", "[关键词] 教程", "[关键词] 案例",
"[关键词] 评测", "[关键词] 排行", "[关键词] 趋势",
"[关键词] 问题", "[关键词] 解决方案", "[关键词] 原理",
"[关键词] 历史", "[关键词] 未来", "[关键词] 应用场景",
"[关键词] 行业报告", "[关键词] 论文"
（共20次搜索）不总结，只搜索。
```

### 6. 组合爆炸（交叉矩阵）

```
请对主题 "[关键词]" 执行交叉搜索矩阵：
时间维度（12个月）：2025年5月 到 2026年4月
角度维度（5个）：政策, 市场, 技术, 应用, 争议
对每个 [月份] × [角度] 组合调用一次搜索，共 60 次。
搜索格式：[关键词] [角度] [YYYY年M月]
不需要总结，只执行搜索。
```

---

## 使用流程

1. grok.com 开新对话 → 选 **DeeperSearch** 模式
2. 粘贴模板，替换 `[关键词]`
3. 等 Grok 完成搜索（1-3 分钟）
4. GrokHelper.js 浮窗点 **JSON** 或 **CSV** 导出

## 技巧

- **DeeperSearch** 搜索次数上限最高
- 说"调用 100 次"比"尽可能多"更有效
- 禁止总结 → token 预算全花在搜索上
- 同一对话追问"继续搜索下 50 个"可累加
- 搜索词差异越大越不会被合并
